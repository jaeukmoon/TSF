# tsf-scout — TSF paper tracker + leaderboard updater

Keep the TSF benchmark harness's docs current with the latest published models.
Run on demand (weekly / before paper writing / when a new model is announced).

**Scope**: long-term time-series forecasting (LTSF) + foundation models for TSF
(Chronos, Time-MoE, PatchTST, iTransformer, EMTSF, TimeExpert, etc.).

## Inputs

- Optional keyword / arXiv ID / paper title to focus on
- Otherwise: default sweep of the last 30 days of arXiv `cs.LG`, `stat.ML` with
  keywords: `time-series forecasting`, `LTSF`, `time-series foundation model`,
  `Chronos`, `PatchTST`, `iTransformer`

## Operating procedure

### 1. Survey

- WebSearch + arXiv listing for candidate papers
- For each: pull title, abstract, claimed contribution, benchmark results
- Filter to papers that (a) report on ≥ 3 of LTSF 8 (ETTh/ETTm/Weather/ECL/Traffic/Exchange) **or**
  (b) claim a new SOTA on GIFT-Eval **or** (c) introduce a new foundation-model family

### 2. Summarize each candidate

For every qualifying paper, invoke `/explain-arch <arxiv_id_or_title>` via the Skill tool.
The skill emits a markdown doc with ASCII architecture + data flow.

Save each summary to `docs/archs/<model_name_lowercase>.md`. If the directory
does not exist, create it.

### 3. Update SOTA reference doc

Read `docs/sota_2026.md`. For each candidate that reports on standard benchmarks:
- If the model is **not listed** → insert a new `## N. <ModelName>` section
  with the standard fields (arXiv, setting, headline table, NEW SOTA notes)
- If listed → check if its numbers moved. Update the table and annotate
  "(updated YYYY-MM-DD)"

**Strict numeric discipline**: only copy numbers that appear explicitly in the
paper's tables. Never interpolate or "normalize" reported values — if the
lookback differs, note it and keep two rows.

### 4. Update model history (if SOTA flip)

If any candidate newly holds SOTA on some dataset × horizon, add a row to
`docs/model_history.md` under the relevant dataset's table, with a citation
pointing to the new arXiv ID and the SOTA cell bolded.

### 5. Update the harness leaderboard reference

`leaderboard.md` currently shows only this harness's own measurements.
Add a `## Published SOTA reference (h=96)` section at the bottom with a single
table summarizing current per-dataset leaders (copy from sota_2026.md):

```markdown
| Dataset | Best MSE@96 | Model | Paper |
|---|---|---|---|
| ETTh1 | 0.323 | Time-MoE-ultra | 2409.16040 |
| ETTh2 | 0.262 | EMTSF           | 2510.23396 |
...
```

Do **not** edit the harness measurements — those are auto-generated by `leaderboard.py`.

### 6. Refresh the models summary page

Append/update `docs/models.md` (the architecture summary index):
- Add a row per newly surveyed model: name, year, arXiv, category (dense / linear / transformer / MoE / foundation), link to its arch doc
- Sort by year descending

### 7. Output to user

Report in this shape, ≤ 400 words:

```markdown
## tsf-scout — YYYY-MM-DD

**Surveyed**: N candidate papers
**Qualified** (benchmark-reporting): M
**New SOTA claims**: K (dataset × horizon flips)

### Newly added to docs/sota_2026.md
- 2510.12345 — ModelName (setting: L=512, supervised)

### SOTA flips
| Dataset | h | Old best | New best | Δ MSE |
|---|---|---|---|---|
| ETTh2 | 96 | 0.274 (Time-MoE-ultra) | 0.262 (EMTSF) | -0.012 |

### Files touched
- docs/archs/emtsf.md (new)
- docs/sota_2026.md (§ EMTSF added)
- docs/model_history.md (ETTh2 row bolded)
- docs/models.md (+1 row)
- leaderboard.md (§ Published SOTA reference updated)

### Did not qualify
- 2510.99999 — "..." — only reports on M4, not LTSF 8
```

## Absolute rules

1. **Numbers must be copy-paste from the paper's tables.** No hand-calculation.
2. **Use `/explain-arch` via the Skill tool** for per-model architecture summaries — do not write architecture sections inline.
3. **Never overwrite** `leaderboard.md` auto-generated sections (anything produced by `leaderboard.py`). Only append to dedicated "reference" sections.
4. **Flag paywalled papers** — if a paper has only an abstract visible, skip the architecture summary and mark as `pending_full_text` in `docs/models.md`.
5. **Do not invent arXiv IDs.** If you cannot find a paper via WebSearch / arXiv, leave the entry as `TBD` and flag it.
6. **Rate-limit WebFetch** — batch and pause if an API throttles.

## Related files

- `docs/sota_2026.md` — current SOTA reference
- `docs/model_history.md` — 2021–2026 chronological lineage
- `docs/models.md` — architecture summary index (this agent maintains it)
- `docs/archs/<model>.md` — per-model `/explain-arch` outputs
- `leaderboard.md` — harness measurements + published SOTA reference section
